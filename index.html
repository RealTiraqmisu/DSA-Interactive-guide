Course Introduction

Why Python for Data Structures & Algorithms?

  • Focus on Logic: Python allows you to write abstract algorithms without worrying about raw memory address management, segmentation faults, and manual buffer reallocations.
  • Rich Built-in Types: Python lists, tuples, and dictionaries are powerful natively-implemented data structures.
  • Readability & Speed: Python code reads like pseudo-code. It will enable you to prototype, test, and implement advanced algorithms faster.

Core Structural Contrast

Feature C Language Python Language
Typing Static (Explicitly declared) Dynamic (Inferred at runtime)
Memory Manual (malloc / free) Automatic Garbage Collection
Braces Curly braces {} & semicolons ; Whitespace & Indentation
Arrays Fixed size, contiguous memory Dynamic size list, polymorphic
OOP Not native (structs only) Native Class & Objects

Module 1: Python Basics for C Programmers

Learn how basic programming concepts map from C syntax to Python semantics.

In C, variables are bound to a type at compile-time (static typing). Memory size is fixed.

In Python, variables are names pointing to dynamic objects. You do not declare types, and a variable can reference different types of objects during execution.

C Code
int age = 20;
double gpa = 3.85;
char grade = 'A';
// Types cannot be changed!
age = "twenty"; // COMPILE ERROR!
Python Code
age = 20         # Inferred as int
gpa = 3.85       # Inferred as float
grade = 'A'      # Inferred as str
# Re-binding is perfectly legal
age = "twenty"   # Now refers to a string
Pro-Tip: In Python, everything is an object. Numbers, strings, and functions are all objects. The variable name is just a pointer!

This is the most critical shift. In C, you used raw contiguous arrays. Python introduces rich native collections:

Type C Equivalent Syntax Example Key Characteristics
String (str) char[] (Null-terminated) s = "Hello DSA" Immutable (cannot modify elements directly). Rich string methods.
List (list) int* with manual realloc fruits = ["apple", 123] Mutable dynamic array. Can hold elements of different types.
Tuple (tuple) Read-only Struct or Array point = (10, 20) Immutable sequence. Used for fixed records or multiple return values.
Dictionary (dict) Custom Hash Map implementation grades = {"Anna": A, "Ben": B} Mutable key-value mapping. Near O(1) lookups.

In C, code blocks are delimited by curly braces {}. In Python, blocks are defined strictly by indentation (typically 4 spaces) and preceded by a colon :.

C Code
if (gpa >= 3.50) {
    printf("Honor Roll\n");
} else if (gpa >= 2.00) {
    printf("Pass\n");
} else {
    printf("Fail\n");
}

for (int i = 0; i < 5; i++) {
    printf("%d\n", i);
}
Python Code
if gpa >= 3.50:
    print("Honor Roll")
elif gpa >= 2.00:  # Note 'elif' instead of 'else if'
    print("Pass")
else:
    print("Fail")

# Range loops
for i in range(5): # range(5) yields 0, 1, 2, 3, 4
    print(i)

Looping through collections: C vs Python

C: Looping by index
int arr[] = {10, 20, 30};
int n = sizeof(arr)/sizeof(arr[0]);
for (int i = 0; i < n; i++) {
    printf("Val: %d\n", arr[i]);
}
Python: Looping directly over collection
arr = [10, 20, 30]
# Directly fetch elements!
for val in arr:
    print(f"Val: {val}")

# Need indexes? Use enumerate()
for index, val in enumerate(arr):
    print(f"Index {index} has value {val}")

C does not have try-catch blocks. Standard error handling involves checking return codes (e.g., if a pointer returned by malloc is NULL, or if fopen returns NULL).

Python employs structural Exception Handling via try-except-finally. This keeps critical algorithms clean of constant nested checks.

C Error Checking
FILE *f = fopen("data.txt", "r");
if (f == NULL) {
    perror("Error opening file");
    return -1;
}
// Read code...
fclose(f);
Python Exception Handling
try:
    with open("data.txt", "r") as f:
        content = f.read()
except FileNotFoundError:
    print("Error: The file does not exist!")
except IOError:
    print("Error: General reading error!")

Module 2: Recursion & Memory Call Stack

How functions call themselves, and how memory manages recursive frames.

Understanding Recursion

A recursive function is simply a function that calls itself. In DSA, recursion is critical for traversing non-linear structures like Trees and Graphs.

Every correct recursive function needs two components:

  1. Base Case(s): The simplest input configuration where the result is computed immediately, ending the recursive chain. Without this, you get an infinite loop resulting in a Stack Overflow (in Python, a RecursionError).
  2. Recursive Step: Reducing the current problem parameters so they move closer to the base case, and calling the function again with those reduced parameters.

In Python, the default recursion limit is 1000 frames (to prevent system memory crash). In C, the limit is based on your OS stack frame memory size.

C vs Python Recursion Syntax

C: Factorial
int factorial(int n) {
    if (n <= 1) { // Base Case
        return 1;
    }
    // Recursive Step
    return n * factorial(n - 1);
}
Python: Factorial
def factorial(n):
    if n <= 1: # Base Case
        return 1
    # Recursive Step
    return n * factorial(n - 1)

Module 3: Complexity Analysis & timeit Module

Evaluate code performance theoretically using Big-O, and empirically using Python's timing tools.

Theoretical Complexity: Big-O

Big-O notation describes how execution time or space requirement scales relative to the input size N in the worst-case scenario.

In Python, collections have highly optimized underlying implementations that dictate different operations' efficiency:

  • List index access (lst[i]): O(1) - contiguous array lookups.
  • List append (lst.append(x)): Amortized O(1) - over-allocates space.
  • List search (x in lst): O(N) - linear scan.
  • Dict lookup (dict[key]): Average O(1) - hash table buckets.
  • Dict insertion (dict[key] = val): Average O(1).
Because dict search is O(1) while list search is O(N), selecting the right data structure dramatically improves algorithm performance!

Module 4: Transition to Object-Oriented Programming (OOP)

Re-organize data structures from C Structs to Python Classes, understanding the mechanics of self.

From Structs to Classes

In C, code and data are separate. You define a struct containing only variables (fields), and write standard functions that accept pointers to that struct to modify them.

In Python OOP, a Class bundles data (attributes) and behavior (methods) together into a single blueprint.

Demystifying self:
In Python class methods, the first parameter is always self. What is it?
It is simply the pointer/reference to the active object instance. In C, you explicitly write print_student(&s). In Python, you write s.print_student(). Under the hood, Python passes the address of s as the first argument, bound to the keyword self!

C Struct vs. Python Class Syntax

C Struct & Function
struct Point {
    int x;
    int y;
};

void move(struct Point *p, int dx, int dy) {
    p->x += dx;
    p->y += dy;
}

// Usage
struct Point pt = {10, 20};
move(&pt, 5, -5);
Python Class & Method
class Point:
    def __init__(self, x, y):
        self.x = x # Attributes
        self.y = y

    def move(self, dx, dy): # self acts as ptr
        self.x += dx
        self.y += dy

# Usage
pt = Point(10, 20)
pt.move(5, -5)
